## Overview

Sequence guard-mechanism verification (on a representative plugin-loader substrate) before vendoring, so the guard form Req 6 ships — dynamic context injection or prose-paragraph fallback — is resolved for the plugin-bundled code path (not the user-level loader). Vendor android-cli files verbatim from `github.com/android/skills@v0.0.2`, then apply the CFA-PATCH marker and guard as a distinct follow-on task. HOW-TO-SYNC edits are serialized (same file); all serialized tasks re-anchor on heading text, not line numbers, since each task shifts downstream offsets. Task 8 owns the `android-cli` cross-reference in `### Covered skills` (Task 7 only creates the bare listing) so no dangling pointer survives a partial completion. Validator, sibling-skill no-touch diff (against a captured baseline ref), and a final behavioral smoke check for the guard close the loop — the lexical ACs gate file state; the behavioral smoke check is the one non-lexical confirmation that the advisory guard is actually honored.
