## Token Usage Rules

Use these semantic tokens for every color, spacing, and typography decision. Never use raw values.

| Token class | Use for | Never use for |
|-------------|---------|---------------|
| `bg-brand-primary` | Primary CTA backgrounds, active nav items | Page backgrounds, body text |
| `bg-surface` | Page backgrounds, main content areas | Raised cards, overlays |
| `bg-surface-raised` | Cards, sidebars, elevated panels | Page backgrounds |
| `bg-surface-overlay` | Modals, dropdowns, tooltips | Page or card backgrounds |
| `text-text-primary` | Body copy, headings, labels | Captions, helper text |
| `text-text-secondary` | Captions, helper text, metadata | Primary content |
| `text-text-disabled` | Disabled control labels | Any interactive element |
| `text-on-brand` | Text on brand-colored backgrounds | Text on light surfaces |
| `border-border` | Default dividers, input borders | Emphasized separators |
| `border-border-strong` | Focused inputs, emphasized dividers | Default borders |
| `bg-error-surface` | Error message backgrounds | Warnings, success states |
| `text-error` | Error text, destructive action labels | Warnings, general copy |
| `bg-success-surface` | Success toast backgrounds | Error or warning states |
| `bg-warning-surface` | Warning banner backgrounds | Error or success states |

## Component Composition Rules

Use shadcn/ui components as the composition layer. Do not build custom primitives when a shadcn component exists.

**Buttons:**
- `variant="default"` — primary actions (one per page section maximum)
- `variant="outline"` — secondary actions
- `variant="ghost"` — tertiary, icon-only, or nav actions
- `variant="destructive"` — irreversible destructive actions only

**Layout:**
- Use `<Card>` for content containers — never a bare `<div>` with manual `p-*` and `border`
- Use `<Separator>` for visual dividers — never `<hr>` or `border-t` on arbitrary elements
- Use `<Sheet>` for slide-over panels, `<Dialog>` for modal overlays — never custom fixed-position divs

**Forms:**
- Wrap every input in `<FormField>` — never a bare `<input>` or `<select>` without a label
- Use `<FormMessage>` for validation errors — never inline `<p className="text-red-*">`
- Use `<Select>` for dropdowns — never `<select>` with manual styling

**Never:**
- Do not apply `rounded-*` directly to shadcn components that have their own radius tokens
- Do not wrap `<Button>` in `<a>` — use the `asChild` prop with `<Link>` instead
- Do not use `<Tooltip>` without a `<TooltipProvider>` ancestor

## Forbidden Patterns

These patterns are explicitly prohibited. The linter will flag them; fix at generation time.

| Forbidden | Use instead |
|-----------|-------------|
| `bg-white`, `bg-black` | `bg-surface`, `bg-text-primary` |
| `text-black`, `text-white` | `text-text-primary`, `text-on-brand` |
| `bg-gray-*`, `text-gray-*`, `border-gray-*` | Surface, text, or border semantic tokens above |
| `bg-blue-*`, `text-blue-*` (or any raw Tailwind color) | `bg-brand-primary`, `text-brand-primary` |
| Arbitrary values: `p-[13px]`, `text-[#ff0000]`, `w-[342px]` | Spacing scale, text scale, or layout tokens |
| Inline style for color: `style={{ color: '#333' }}` | `text-text-primary` class |
| Inline style for spacing: `style={{ padding: '12px' }}` | `p-3` (spacing scale) |
| Raw hex or rgb in CSS: `color: #ffffff` | `color: var(--color-text-primary)` |
| `font-size: 14px` or `text-sm`-style raw values in CSS | `font-size: var(--text-sm)` |
