# /ui-brief

Generates a `DESIGN.md` constraint document and a Tailwind v4 `@theme` semantic token starter for a new project. Run once at project setup to establish the design system before any UI components are built.

## Step 1: Gather Inputs

If `project-name` was not passed as an argument, prompt: "What is the project name?"

If `brand-hue` was not passed as an argument, prompt: "What is your primary brand color hue? (0–360 oklch — 0/360=red, 30=orange, 60=yellow, 120=green, 210=sky, 260=violet, 300=pink)"

If `brand-lightness` was not passed, use the default `medium`. Do not prompt — only override if the user explicitly provides it.

Store all three values. Proceed to Step 2 without outputting anything yet.

## Step 2: Detect Conventions

Run the following detections silently. Do not output results yet — report findings in Step 5.

**CSS file location** (check in this order):
1. `app/globals.css` — Next.js App Router
2. `src/styles/globals.css` — Vite / React
3. `src/index.css` — Vite fallback
4. If none found: default to `app/globals.css` (will need to be created); note "CSS file not found — will create app/globals.css"

**DESIGN.md location**:
- If `frontend/` directory exists at project root: write to `frontend/DESIGN.md`
- Otherwise: write to `DESIGN.md` at project root

**Tailwind version**:
- Read `package.json` and find `tailwindcss` in dependencies or devDependencies
- If version starts with `4` or `^4`: set `tailwind-version = v4`
- If version starts with `3` or `^3`: set `tailwind-version = v3`; note "Tailwind v3 detected — @theme block requires v4; will generate tailwind.config.ts token section instead"
- If not found: default to `tailwind-version = v4`; note "tailwindcss not found in package.json — assuming v4"

## Step 3: Generate DESIGN.md

**Idempotency check**: Read the DESIGN.md target path.
- If file does not exist: proceed to write.
- If file exists and first line is `<!-- ui-brief:generated -->`: file was previously generated by this skill. Skip writing; record "DESIGN.md: skipped (already generated)" for Step 5 output.
- If file exists but does NOT start with `<!-- ui-brief:generated -->`: this is a hand-authored DESIGN.md. Ask the user: "DESIGN.md already exists and was not generated by /ui-brief. Overwrite it? (yes/no)" If no, skip and record "DESIGN.md: skipped (user declined)". If yes, proceed to write.

Read `${CLAUDE_SKILL_DIR}/references/design-md-template.md` for the exact file content to write to the DESIGN.md target path.

## Step 4: Generate @theme Block

**Idempotency check**: Read the detected globals.css file.
- If it contains `/* ui-brief:generated-start */`: the block was previously written. Skip; record "globals.css @theme: skipped (already generated)" for Step 5 output. Note: if the project's generated block predates dark mode support (no `@media (prefers-color-scheme: dark)` block present), instruct the user to remove the `/* ui-brief:generated-start */` through `/* ui-brief:generated-end */` block from the CSS file and re-run `/ui-brief` to generate adaptive theming.
- If file does not exist: create it.
- Otherwise: append the block below after the last existing rule (or at end of file).

Read `${CLAUDE_SKILL_DIR}/references/theme-template.md` for the brand color derivation table and the exact CSS block to append to globals.css. Substitute all `{...}` variables with derived values before writing.

## Step 5: Confirm Output

Output the following confirmation. Keep total output under 2000 tokens — show token names and paths, not file contents.
